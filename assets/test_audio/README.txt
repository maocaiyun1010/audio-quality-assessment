音源目录说明（自动扫描生成播放顺序）
=====================================

1. 支持的格式：wav / mp3 / m4a / flac / ogg / aac（扩展名不区分大小写）

2. 目录结构（二选一）：
   - 推荐：在一级子文件夹中按类存放（如 曲艺、语声），文件夹名将作为报告里的「分组」。
     播放顺序 = 子文件夹名「自然排序」后，每个文件夹内文件名「自然排序」
     （例如 quyi_2.wav 排在 quyi_10.wav 前）。
   - 仅当「没有任何子文件夹内含音源」时，才使用根目录下的音频，分组为「根目录」。
     （只要任一子文件夹里有音源，根目录下的 wav/mp3 会被忽略，避免与子目录重复。）

3. 忽略：以点开头的文件/文件夹；非上述扩展名的文件（如本 README）。

4. 试跑占位文件生成（可选）：
   python tools\gen_placeholder_wavs.py

5. 每次采集会在 output\recorded 下写入「会话前缀_playlist.json」，
   与本次录音一一对应，供评分与 Word 报告还原音源顺序与分组。

6. 每条有效录音默认 30 秒（环境变量 SPEAKER_PER_TRACK_SEC 或 run_all.py --seconds）。
   若曾设置旧环境变量导致过短，请清除 SPEAKER_PER_TRACK_SEC 后重跑。

7. 录不到喇叭声时：运行 python self_test.py 查看麦克风列表，设置
   SPEAKER_INPUT_DEVICE=设备索引 或 名称子串。
